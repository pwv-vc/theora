# security
- Use `execFileSync(cmd, [arg1, arg2, ...])` with an array of arguments for all shell command execution — never use `execSync` with a string-interpolated command, which passes through `/bin/sh -c` and is vulnerable to shell injection. Confidence: 0.95
- Add a `safeJoin(base, untrusted)` utility in `src/lib/paths.ts` that calls `resolve()` and verifies the result starts with `resolve(base) + '/'` — use it everywhere user-controlled or LLM-controlled input is joined into a file path. Confidence: 0.90
- Always run LLM-generated slugs (e.g., concept slugs from JSON) through `slugify()` before using them in file paths — LLM output is untrusted and can contain `../` traversal sequences. Confidence: 0.90
- Strip directory components from browser-supplied filenames with `basename(file.name)` before using in `path.join()` — `File.name` is fully attacker-controlled and `path.join()` does not prevent traversal. Confidence: 0.95
- Block SSRF in `fetchUrl()` by resolving the hostname with `dns.lookup()` and rejecting private/loopback IP ranges (127.x, 10.x, 172.16-31.x, 192.168.x, 169.254.x) before fetching — fail closed if DNS resolution fails. Confidence: 0.85
- Apply `MAX_FILE_SIZE` to URL fetch responses (not just file uploads) — stream with a byte counter and abort if the response exceeds the limit. Confidence: 0.85
- For LLM-generated Python chart execution: add a 30-second timeout to `execFileSync`, validate imports against an allowlist (matplotlib, math, datetime, etc.), and block `subprocess`, `os.system`, `exec`, `eval` with a regex scan before writing and executing the code. Confidence: 0.85
- Escape double quotes and newlines in user input before embedding in YAML frontmatter string fields — raw interpolation breaks YAML structure (e.g., `title: "${question}"` where question contains `"`). Confidence: 0.85
- Add a `SECURITY.md` to the repo documenting: prompt injection risk from untrusted documents, chart code execution risk, SSRF risk, and the recommendation to only ingest trusted content. Confidence: 0.80
- Sanitize all server-side markdown rendering with `sanitize-html` (after `marked.parse()`) and all client-side markdown rendering with `DOMPurify.sanitize()` before assigning to `innerHTML` — never assign raw LLM or user-supplied HTML directly. Confidence: 0.90
- Use `textContent` (per-cell DOM construction) instead of `row.innerHTML` template literals when building table rows from untrusted data — `innerHTML` with string interpolation is an XSS vector even for log/stats data. Confidence: 0.90
- Pin CDN `<script>` tags to exact semver versions and add `integrity="sha384-..."` + `crossorigin="anonymous"` SRI attributes — compute hashes with `curl -sL <url> | openssl dgst -sha384 -binary | openssl base64 -A`. Confidence: 0.85
- Wrap untrusted source content in XML-style delimiters (e.g., `<source_content>...</source_content>`, `<wiki_articles>...</wiki_articles>`) in LLM prompts and add an explicit instruction to treat content inside those tags as data, not commands — this is the primary prompt injection mitigation. Confidence: 0.85
- Validate URL route parameters (`:slug`, `:type`) against a strict allowlist regex (e.g., `^[a-z0-9][a-z0-9-]*$`) before using them in `path.join()` — Hono/Express route params are not sanitized and can contain `..` traversal sequences. Confidence: 0.90
- Set mermaid `securityLevel: 'strict'` (not `'loose'`) and HTML-escape diagram source text before inserting into the DOM — `securityLevel: 'loose'` allows arbitrary HTML/JS execution inside diagrams. Confidence: 0.85
